# Repository Layout

`standards/REPO_LAYOUT.yaml` is now the authoritative repository layout contract.

Use it for:

- architecture zones
- authoritative roots
- placement verification
- zone dependency policy

Use the CLI for live state instead of treating this file as a hand-maintained tree:

- `saguaro architecture map`
- `saguaro architecture verify`
- `saguaro architecture zones`
- `saguaro architecture explain <path>`
